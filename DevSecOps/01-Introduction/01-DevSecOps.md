# DevSecOps Notes

## What is DevSecOps?
DevSecOps = Development + Security + Operations

It integrates **security practices into every stage of the DevOps lifecycle**.

---

## Why DevSecOps?
- Traditional security is done at the end (too late)
- DevSecOps shifts security earlier (Shift Left)
- Reduces vulnerabilities early
- Improves compliance and reliability

---

## Key Principles
- Shift Left Security
- Automation of security checks
- Continuous monitoring
- Collaboration between Dev, Sec, Ops teams

---

## DevSecOps Lifecycle
1. Plan
2. Code
3. Build
4. Test
5. Release
6. Deploy
7. Operate
8. Monitor

(Security is integrated at each stage)

---

## Tools Used
- SAST: SonarQube, Checkmarx
- DAST: OWASP ZAP, Burp Suite
- SCA: Snyk, Black Duck
- Container Security: Trivy, Anchore
- CI/CD: Jenkins, GitHub Actions

---

## Benefits
- Early vulnerability detection
- Faster delivery
- Reduced cost of fixing issues
- Improved security posture
