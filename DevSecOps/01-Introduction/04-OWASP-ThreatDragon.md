# OWASP Threat Dragon Notes

## What is OWASP Threat Dragon?
An open-source tool for **threat modeling**.

Used to:
- Create architecture diagrams
- Identify threats using STRIDE
- Document security risks

---

## Key Features
- Easy-to-use UI
- DFD diagram creation
- STRIDE-based threat identification
- JSON-based model storage
- Web + Desktop versions

---

## Components in Threat Dragon
- Process
- Data Store
- External Entity
- Data Flow

---

## How It Works
1. Create a new model
2. Draw architecture (DFD)
3. Add components
4. Identify threats
5. Assign severity
6. Add mitigations

---

## Example Threat
- Data flow without encryption → Information Disclosure

Mitigation:
- Use HTTPS / TLS

---

## Benefits
- Visual threat modeling
- Easy collaboration
- Improves security design early

---

## Use Case in DevSecOps
- Integrate during design phase
- Combine with Shift Left strategy
- Helps in secure architecture planning
